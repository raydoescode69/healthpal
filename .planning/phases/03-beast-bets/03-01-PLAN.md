Create `lib/betsService.ts` with all Beast Bets business logic. Import `supabase` from `../lib/supabase` and types from `../lib/types`.

**createChallenge(userId: string, data: { title: string, metric: Challenge['metric'], target: number }): Promise&lt;Challenge&gt;**
- Calculate `expires_at` as next midnight UTC: `new Date(Date.now())`, set hours to 23, minutes 59, seconds 59 in UTC. Use `toISOString()`.
- INSERT into `challenges` table with `creator_id: userId`, spread data fields, `status: 'active'`, `expires_at`.
- After challenge creation, INSERT the creator as the first participant into `challenge_participants` with `challenge_id`, `user_id: userId`, `current_value: 0`.
- Return the created challenge. Throw on error.

**joinChallenge(userId: string, challengeId: string): Promise&lt;ChallengeParticipant&gt;**
- First SELECT the challenge to verify it exists and `status === 'active'`.
- If not active, throw `Error("Challenge is no longer active")`.
- INSERT into `challenge_participants` with `challenge_id: challengeId`, `user_id: userId`, `current_value: 0`.
- Handle unique constraint error gracefully (user already joined) вҖ” return existing participant row instead.
- Return the participant record.

**getInviteLink(challengeId: string): string**
- Return `healthpal://bets/join/${challengeId}` вҖ” this is the deep link format for expo-router.
- Note: Deep link handling will use expo-router's built-in linking config.

**shareInvite(challengeId: string, challengeTitle: string): Promise&lt;void&gt;**
- Import `* as Sharing` from `expo-sharing` and check `Sharing.isAvailableAsync()`.
- If available, call `Sharing.shareAsync` is for files only. Instead, use the `Share` API from `react-native`:
  ```typescript
  import { Share } from 'react-native';
  await Share.share({
    message: `Join my Beast Bet: "${challengeTitle}"! Can you beat me? ${getInviteLink(challengeId)}`,
  });
  ```
- If Share is not available, log a warning.

**loadUserChallenges(userId: string): Promise&lt;Challenge[]&gt;**
- SELECT from `challenges` WHERE `creator_id = userId` OR `id IN (SELECT challenge_id FROM challenge_participants WHERE user_id = userId)`.
- Order by `created_at DESC`.
- Return array.

**loadChallengeDetail(challengeId: string): Promise&lt;{ challenge: Challenge, participants: (ChallengeParticipant & { display_name?: string, avatar_emoji?: string })[] }&gt;**
- SELECT challenge by id.
- SELECT all participants for this challenge_id.
- For each participant, try to fetch display name from `profiles` table (SELECT name FROM profiles WHERE id = participant.user_id). If profiles table doesn't have name, use "Player" + index as fallback.
- Add `avatar_emoji` field: assign deterministic emoji from a small array `['рҹҗү','рҹҰҒ','рҹҗә','рҹҰ…','рҹҗ»','рҹҰҠ','рҹҗҜ','рҹҰҲ']` based on participant index.
- Sort participants by `current_value` DESC and assign rank.
- Return challenge + enriched participants.

**subscribeToChallenge(challengeId: string, onUpdate: (participants: ChallengeParticipant[]) => void): () => void**
- Create a Supabase Realtime channel: `supabase.channel(`challenge-${challengeId}`)`.
- Subscribe to postgres_changes on `challenge_participants` table filtered by `challenge_id=eq.${challengeId}` for UPDATE events.
- On each change, call `loadChallengeDetail(challengeId)` to get fresh sorted participants, then call `onUpdate` with the participants array.
- Return an unsubscribe function that calls `supabase.removeChannel(channel)`.
- CRITICAL (from PITFALLS.md): This unsubscribe function MUST be called in component cleanup to prevent connection leaks.

**resolveExpiredChallenges(userId: string): Promise&lt;void&gt;**
- SELECT from challenges WHERE `status = 'active'` AND `expires_at < NOW()` AND (`creator_id = userId` OR challenge has participant with user_id).
- For each expired challenge:
  - SELECT participants ORDER BY current_value DESC.
  - The participant with highest current_value is the winner.
  - UPDATE challenge SET `status = 'completed'`, also store winner info in a comment or separate update.
  - Note: We don't have a winner_id column from Phase 1 schema, so we determine winner client-side from sorted participants. The status change to 'completed' is what matters.

**updateParticipantProgress(participantId: string, newValue: number): Promise&lt;void&gt;**
- UPDATE `challenge_participants` SET `current_value = newValue` WHERE `id = participantId`.
- This triggers the realtime subscription for other clients.

Implementation:
- `loadChallenges`: call `betsService.loadUserChallenges`, set challenges array, also call `resolveExpired` to clean up any stale active ones.
- `loadChallenge`: call `betsService.loadChallengeDetail`, set `activeChallenge` and `participants`.
- `createChallenge`: call `betsService.createChallenge`, then reload challenges. Return the created challenge so the UI can navigate to it or trigger sharing.
- `joinChallenge`: call `betsService.joinChallenge`, then `loadChallenge` to refresh.
- `updateProgress`: call `betsService.updateParticipantProgress`.
- `subscribeToActive(challengeId)`: First call `get().unsubscribeFromActive()` to clean up any prior subscription. Then call `betsService.subscribeToChallenge(challengeId, (newParticipants) => set({ participants: newParticipants }))`. Store the returned unsubscribe function in state.
- `unsubscribeFromActive()`: If `get().unsubscribe` exists, call it and set `unsubscribe: null`.
- `resolveExpired`: call `betsService.resolveExpiredChallenges`.
- `clearBets`: reset all state, call unsubscribeFromActive first.
