2. Create `supabase/migrations/001_viral_features_tables.sql` with all 4 tables:

   **aura_scores table:**
   - id UUID PK default gen_random_uuid()
   - user_id UUID NOT NULL references auth.users(id) ON DELETE CASCADE
   - score INTEGER NOT NULL (0-100)
   - sleep_score INTEGER (0-100)
   - nutrition_score INTEGER (0-100)
   - movement_score INTEGER (0-100)
   - hydration_score INTEGER (0-100)
   - label TEXT NOT NULL (THRIVING, RESTING, PUSHING, STRUGGLING)
   - scored_at DATE NOT NULL DEFAULT CURRENT_DATE
   - created_at TIMESTAMPTZ DEFAULT now()
   - UNIQUE(user_id, scored_at) -- one score per user per day
   - RLS: users can SELECT/INSERT/UPDATE own rows (auth.uid() = user_id)

   **roasts table:**
   - id UUID PK default gen_random_uuid()
   - user_id UUID NOT NULL references auth.users(id) ON DELETE CASCADE
   - roast_text TEXT NOT NULL
   - verdict_title TEXT NOT NULL
   - verdict_emoji TEXT NOT NULL
   - calories INTEGER
   - steps INTEGER
   - sleep_hours REAL
   - scored_at DATE NOT NULL DEFAULT CURRENT_DATE
   - created_at TIMESTAMPTZ DEFAULT now()
   - UNIQUE(user_id, scored_at)
   - RLS: users can SELECT/INSERT own rows

   **challenges table:**
   - id UUID PK default gen_random_uuid()
   - creator_id UUID NOT NULL references auth.users(id) ON DELETE CASCADE
   - title TEXT NOT NULL
   - metric TEXT NOT NULL (steps, calories, workout_minutes)
   - target INTEGER
   - status TEXT NOT NULL DEFAULT 'active' (active, completed, cancelled)
   - created_at TIMESTAMPTZ DEFAULT now()
   - expires_at TIMESTAMPTZ NOT NULL
   - RLS: authenticated users can SELECT challenges they created OR are a participant in. Creator can INSERT/UPDATE own challenges.

   **challenge_participants table:**
   - id UUID PK default gen_random_uuid()
   - challenge_id UUID NOT NULL references challenges(id) ON DELETE CASCADE
   - user_id UUID NOT NULL references auth.users(id) ON DELETE CASCADE
   - current_value INTEGER DEFAULT 0
   - joined_at TIMESTAMPTZ DEFAULT now()
   - UNIQUE(challenge_id, user_id)
   - RLS: participants can SELECT rows for challenges they are in. Users can INSERT themselves. Users can UPDATE own rows.

   Enable RLS on all 4 tables. Create indexes on user_id and scored_at columns.
